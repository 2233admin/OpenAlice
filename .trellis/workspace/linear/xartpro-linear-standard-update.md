Created the first explicit Linear operating standard for agent-managed projects:

- Document: `XartPro Linear Operating Standard v1`
- Attached to: `XartPro Linear 模板`

Core rule:

- classify task complexity before creating issues
- Class A = quick task
- Class B = feature track
- Class C = complex project

Class C now requires:

- Initiative
- Project
- milestones/phases
- parent program issue
- child issues
- dependency graph
- project update
- handoff issue

This is the rule that should prevent future agents from treating complex rebuilds as loose issue piles.
