## Scope

Turn the OpenAlice Linear structure into a reusable project kit for future XartPro work.

This is not a workspace-wide lock, compliance sweep, or mandatory migration.

## Goal

Other projects should not rebuild the same coordination wheel:

- classify work as Class A / B / C
- reuse the right amount of Linear structure
- copy OpenAlice's project spine only when complexity matches
- give external agents one clear issue, branch, safe zone, forbidden zone, and handoff format
- keep durable design/architecture context in Linear Documents instead of chat logs

## Reference

- Project Reuse Kit: `XartPro Project Reuse Kit v1`
- Reference implementation: `OpenAlice`
- OpenAlice spine: `XAR-709`
- Claude 5090 entry: `XAR-711`

## Work

- Keep the reusable kit current as OpenAlice evolves.
- Add examples from future projects when a pattern repeats.
- For a new complex project, copy the kit shape instead of inventing a fresh issue tree.
- For small projects, intentionally use less structure.

## Acceptance

- New agents can start from the kit and avoid asking the user to re-explain granularity.
- Future complex projects have a faster setup path.
- Small projects are not over-structured.
- OpenAlice remains the concrete example, not a one-off chat artifact.
